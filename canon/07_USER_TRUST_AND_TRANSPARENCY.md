# 07_USER_TRUST_AND_TRANSPARENCY

**Status:** CANONICAL  
**Version:** v0.1

## Purpose
Establishes the compact of trust between the user and the system, prioritizing transparency over seamlessness.

## Hard Constraints
*   **AI Disclosure:** Any content generated or summarized by AI must be explicitly labeled as such (e.g., "AI-Summarized").
*   **Uncertainty Signaling:** If the system is less than 99% confident in a match or cluster, it must display a "Low Confidence" indicator.
*   **Correction Mechanism:** Users must have a visible, low-friction path to report "Bad Cluster," "Wrong Source," or "Broken Link."
*   **Neutral Tone:** System voice (notifications, summaries) must be clinical, concise, and void of emotional adjectives.

## Explicit Non-Goals
*   **No "Seamless" Magic:** We do not hide the seams if hiding them misleads the user about the data's origin.
*   **No Personalized Echo Chambers:** We do not hide conflicting views to comfort the user.
