# 09_PHASE_AND_APPROVAL_MODEL

**Status:** CANONICAL  
**Version:** v0.1

## Purpose
Establishes the governance structure for project lifecycle and decision-making authority.

## Hard Constraints
*   **Phase Gates:** Movement from one phase (e.g., Discovery) to the next (e.g., MVP Build) requires explicit, written approval.
*   **The "Canon Check":** No plan or design can be approved if it violates any active Canon rule (00–08).
*   **Approver Authority:** Only the designated Product Owner (or Lead Architect) can approve Phase advancement.
*   **MVP Definition:** The MVP must satisfy all hard constraints in 00–08 but may have limited scope (e.g., fewer sources, limited topics).

## Explicit Non-Goals
*   **No "Shadow IT":** No features or code should be built outside the approved phase scope.
*   **No "Soft Launches":** "Beta" is a formal phase, not an excuse to ignore Canon.
